# Plantilla 03 — Documentos que nos puedes aportar

**Quién la rellena:** cualquier persona con acceso a la documentación del sistema.
**Cómo:** marca con `[x]` lo que tengas, indica quién lo tiene y adjúntalo (o dinos
que existe). Si no tienes nada documentado, no pasa nada: las plantillas 01 y 02
son suficientes para empezar.

> **Antes de adjuntar nada:** revisa que no haya datos de personas reales. Si los
> hay, anonimiza el documento o no lo envíes y descríbelo con palabras.

## Para qué usamos cada documento

| Documento | Qué nos aporta |
|---|---|
| Capturas de pantalla | Cómo se presenta el resultado y cómo se captura el override (lo que más condiciona la decisión) |
| Ficha técnica / model card | Capacidades, límites y rendimiento que luego comprobamos si se comunican al usuario |
| Onboarding / manual | Si se fijan bien las expectativas desde el principio |
| Diseño / especificación | La intención de diseño, para contrastarla con la realidad de uso |
| Protocolo de uso | El contexto real donde se decide |
| Validación / subgrupos | Si los límites y posibles disparidades se conocen y se vigilan |
| Registros de uso (agregados) | Señales de exceso de confianza, fatiga de alerta, tasa de override |
